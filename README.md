# Customer Development Workspace

## Folders
- `hypotheses/` — filled-out canonical business model / hypothesis docs
- `strategy/` — canonical prioritization and portfolio-shaping docs (for example hypothesis ranking)
- `experiments/` — canonical pass/fail experiment docs plus experiment README
- `interviews/` — interview plans, notes, and transcripts
- `research/` — supporting research memos, design docs, packets, matrices, and open questions
- `deliverables/` — polished deliverables meant to be shared or cited directly
- `memos/` — standalone strategic memos and longer-form internal framing docs
- `frameworks/` — reusable methods and process docs
- `syntheses/` — cross-experiment or sprint-level summaries
- `ops/` — operational / reliability docs and live process notes
- `templates/` — reusable templates and checklists
- `reference/` — source images / references used to build templates

## Canon vs support
- Canonical business model docs live in `hypotheses/`.
- Canonical prioritization / portfolio-shaping docs live in `strategy/`.
- Canonical pass/fail experiments live in `experiments/` and should follow `templates/11-experiment-design.md`.
- Supporting materials do not need to match the experiment template, but should live in the appropriate support folder and be clearly named.
