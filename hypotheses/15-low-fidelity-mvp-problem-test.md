# Low Fidelity MVP Problem Test
> Built from `templates/15-low-fidelity-mvp-problem-test.md`
> Goal: Determine whether people care enough about the problem to act on a low-fidelity entry point
> Scope: Unified startup — testing whether operator-facing content + lightweight web destination can generate meaningful interest in agent coordination / accountability / workflow reliability
> Last updated: 2026-03-23

---

## Purpose

Use the lowest-friction public artifact to test whether real humans care enough about the problem to engage.

This test is **not** primarily about branding polish or final company identity.
It is about whether a problem-led message can generate:
- attention from the right audience
- explicit recognition of the pain
- concrete replies / conversations / pilot interest
- evidence about which surface converts best: personal account, legacy distribution accounts, or a lightweight website CTA

The current decision under test is:
> Should the low-fidelity MVP be primarily a **personal distribution surface** (Jakub’s Twitter + blog), a **team website**, or a hybrid of both?

Current best hypothesis:
> The fastest and highest-signal MVP is **personal-account-led distribution** with a **lightweight follow-up website** as the capture / explanation surface.

Reasoning:
- personal voice is faster to ship and requires no internal consensus loop
- current team brand is unstable / likely to rebrand
- early-stage audiences often respond more to a person with a point of view than to an underdefined company shell
- a website is still useful, but more as a destination and signal-capture layer than as the primary distribution engine

---

## Traffic / Invitation Tactics

### Primary distribution hypothesis
Use **Jakub’s Twitter account** as the main distribution surface for original posts.

Why:
- fastest iteration loop
- strongest authentic voice
- avoids brand/alignment bottlenecks
- lets message testing happen before company identity hardens

### Secondary distribution hypothesis
Use **former brand / product Twitter accounts** only as amplification satellites, not as the main voice.

Rules:
- legacy accounts should repost / quote / cross-promote when audience adjacency is strong
- they should not pretend the old brand is still the core identity if it is not
- they should route attention toward the canonical post / account / destination

### Website hypothesis
Have a **simple follow-up website** that acts as:
- a destination for people who want more context
- a place to capture CTA clicks / email / pilot requests / replies
- a stable link target that can survive Twitter post churn

The website should **not** be treated as a heavy branding exercise yet.
It is an instrumented experiment surface, not a final company homepage.

### Initial channel mix to test
1. personal Twitter post
2. repost / quote from relevant legacy accounts
3. optional direct sends to known relevant operators/builders
4. optional community reposting where audience fit is high
5. lightweight website link as the follow-up destination

## Capture Mechanisms

The low-fidelity MVP should make it easy to capture intent signals, such as:
- request pilot
- book a call / conversation
- reply with current workflow pain
- email capture / waitlist (optional)
- DM path
- simple form: “where do your agents get stuck?”

Best current CTA candidates:
- **Tell me where your agents get stuck**
- **Request a pilot**
- **Show me your workflow**
- **Talk to us / talk to me**

Important distinction:
- **Twitter / blog = top-of-funnel distribution + message test**
- **website = capture + elaboration + attribution surface**

---

## Quantitative Measures

Track at minimum:
- post impressions
- profile clicks
- link clicks
- website visits
- source of traffic (personal account vs legacy account vs other)
- CTA conversion rate
- number of meaningful replies
- number of DMs / intros / pilot requests
- return visits if measurable

### Metrics that matter most
At this stage, prioritize:
1. meaningful replies from relevant operators/builders
2. DMs / intros / requests to talk
3. pilot interest
4. evidence that one framing consistently outperforms others

Do **not** overweight vanity metrics like raw impressions without downstream response quality.

---

## Qualitative Follow-up

For anyone who engages, ask:
- what problem they thought the post/page was describing
- whether this feels like a current pain or just an interesting idea
- where their current agent workflows break down
- whether the harder problem is coordination, visibility, trust, or follow-through
- whether they would want a lightweight connection tool first or a heavier accountability / continuity layer
- whether the message felt more credible coming from a person or would benefit from a stronger company wrapper

Specific qualitative question for this test:
> Did the audience respond because Jakub has a credible personal point of view, or because the startup proposition itself was compelling regardless of who said it?

---

## Interpretation Questions

After running traffic and follow-up conversations, ask:
1. Did people care enough to act?
2. Was Twitter sufficient as the primary distribution surface?
3. Did the follow-up website improve conversion or understanding?
4. Did personal-account-led distribution outperform institution-led messaging?
5. Did legacy account amplification create qualified traffic or just noise?
6. Which pain frame resonated most strongly?
7. Did respondents react to a real recurring workflow problem, or only to interesting writing?
8. Is the fastest next step a better content engine, a clearer landing page, or a stronger brand shell?

## Pass/Fail Planning

### Target audience
Primary audiences:
- founders / operators already using agents
- builders of agent tooling / runtimes / orchestration layers
- crypto-native operators exploring accountable agent workflows
- people frustrated by manual babysitting of agent systems

### Traffic sources
- Jakub’s Twitter account (primary)
- selected legacy product/brand Twitter accounts (secondary amplification)
- direct outreach to warm relevant people (optional support)

### Meaningful interest definition
Meaningful interest includes:
- a reply that clearly maps the message to a real current workflow pain
- a DM or intro request
- a call / meeting / pilot request
- repeated engagement from a relevant operator/builder

### Pass threshold
A useful first pass would be:
- at least several meaningful responses from relevant people
- at least one or more real conversations / pilot discussions
- clear evidence that personal-first distribution generates qualified interest
- evidence that the website helps capture / deepen interest rather than merely existing

### Fail threshold
- posts get attention but almost no qualified responses
- website receives clicks but no real CTA conversion
- legacy account amplification drives low-quality traffic only
- people find the idea interesting but do not connect it to a real pain they currently have

### Core experimental bet
**Pass candidate:**
> Personal Twitter as the main distribution surface + lightweight website as the follow-up destination is enough for a low-fidelity MVP test.

**Fail candidate:**
> The audience requires a stronger institutional wrapper before they will engage seriously.

- [ ] Real potential customers were invited into the MVP
- [ ] Personal-account-led distribution was tested directly
- [ ] Legacy-account amplification was tested selectively where relevant
- [ ] A lightweight follow-up website existed as a destination / capture layer
- [ ] Interest signals were captured and attributed by source
- [ ] Direct feedback was collected from real respondents
- [ ] We learned whether the MVP should remain personal-first or move toward a more explicit team website

## Current recommendation

For the next iteration:
- **Use Twitter as the main distribution surface**
- **Have a follow-up website**
- **Keep the website lightweight and under unilateral decision control**
- **Avoid waiting for team-wide consensus on branding/content before testing message pull**

Practical framing:
> personal voice for distribution, lightweight site for capture, team brand later if the signal justifies institutionalization.
